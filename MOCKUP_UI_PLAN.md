# Daylog UI Plan

> 흔들어 끄는 미라클 모닝 알람 + 해제 직후 굿모닝 브리핑(날씨) 앱의 화면 구성.
> (피벗 전 3탭 운동·식단·알람 mock 구조에서 단일 알람 + 브리핑으로 축소됨.)

---

## 전체 구조

하단 탭 없는 **단일 화면 앱**.

| 화면 | 역할 | 핵심 액션 |
|---|---|---|
| 알람 (홈) | 미라클 모닝 알람 관리 | 알람 추가/수정/On·Off |
| 알람 울림 | 흔들어 해제 | 목표 횟수 채우기 |
| 굿모닝 브리핑 | 해제 직후 오늘 날씨 | 확인 후 시작하기 |

---

## 1. 알람 화면 (홈)

- `Scaffold` + AppBar "알람" + FAB(알람 추가)
- 본문: `AlarmListView` (`TabScaffold` 제목 "알람" / 부제)
- 알람 카드: 큰 시간, 라벨·반복 요일, 흔들기 횟수, On/Off `Switch`
- 스와이프(endToStart) → 삭제
- 카드 탭 → 편집 BottomSheet
- 빈 상태: `EmptyState` ("설정된 알람이 없어요")

### 알람 편집 BottomSheet (`AppBottomSheetScaffold`)
- 시간 표시(탭 → `showTimePicker`)
- 반복 요일: 프리셋(주말만/평일만/전체) + 요일 원형 버튼
- 흔들기 횟수: −/+ 스텝퍼 (5~100, 5단위)
- 저장 버튼(`AppPrimaryButton`)

---

## 2. 알람 울림 화면 (전체화면)

```text
05:30
미라클 모닝
20번 흔들어서 해제
[=========-----------]
7 / 20
[ 흔들림 추가 ]
```

- 큰 현재 시각 + 라벨 + 진행률 바 + 카운트
- 목표 도달 → "해제 완료" → 알람 정지 후 **굿모닝 브리핑으로 `pushReplacement`**
- 좌상단 닫기(x) → 알람만 정지(브리핑 없이 종료, 중단 경로)
- 흔들기 감지는 현재 버튼 mock — 후속에서 `sensors_plus` 연동

---

## 3. 굿모닝 브리핑 화면

```text
좋은 아침이에요
5월 25일 (일) · 05:30

  12°  흐림
  최고 18°  ·  최저 7°

[ 시작하기 ]
```

- 인사 + 오늘 날짜/시각
- 현재 기온 + 날씨 상태(WMO 코드 → 한글) + 당일 최고/최저
- open-meteo 실시간 조회, 위치는 GPS(`geolocator`)
- 상태 분기:
  - 로딩: 스피너
  - 성공: 날씨 카드
  - 위치 권한 거부: 안내 + "권한 허용"(재요청)
  - 영구 거부: 안내 + "설정 열기"
  - 위치 서비스 꺼짐 / 실패: 안내 + "다시 시도"
- "시작하기" → 홈으로

## 컴포넌트 재사용

| 컴포넌트 | 사용 위치 |
|---|---|
| `AppBottomSheetScaffold` | 알람 편집 시트 |
| `AppPrimaryButton` | 시트 저장 / 브리핑 시작하기 / 울림 버튼 |
| `EmptyState` | 알람 빈 상태, 브리핑 권한·실패 상태 |
| `TabScaffold` | 알람 목록 헤더 |
